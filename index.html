Étude et modélisation d’un système d’information pour AVS Immobilier

Résumé :

Ce projet s’est déroulé en deux phases. La première phase, réalisée individuellement, consistait à enquêter auprès d’une entreprise afin de comprendre ses besoins en gestion de données (commandes, facturation, clients, stocks, etc.). J’ai choisi AVS Immobilier, une agence immobilière locale, pour analyser ses processus métiers. J’ai ensuite synthétisé ces informations pour proposer un premier modèle conceptuel de données (MCD) et un dictionnaire de données.

Dans la seconde phase, en binôme, nous avons sélectionné le modèle le plus pertinent du groupe pour le développer concrètement. Nous avons conçu le schéma relationnel, implémenté la base avec SQLite, défini des règles de gestion (prix non négatifs, formats d'e-mails, etc.) et ajouté des données simulées. Enfin, nous avons rédigé une vingtaine de requêtes SQL permettant d’exploiter efficacement les données selon les besoins d’AVS Immobilier.

Compétences développées :

  • Analyse des besoins d’un système d’information réel

  • Modélisation conceptuelle et relationnelle d’une base de données (MCD, dictionnaire de données, schéma relationnel)

  • Implémentation d’une base de données avec SQLite

  • Rédaction de scripts SQL : création, insertion, interrogation

  • Formulation de requêtes pertinentes répondant à des besoins métiers

Langage et environnement utilisés :

  • SQLite3

  • SQL (scripts de création, insertion, interrogation)

  • Outils de modélisation (MCD et dictionnaire de données)

  • Outils bureautiques pour la rédaction du rapport (Word)


Création d’un site web pour le parc de loisirs DinoTerra

Résumé :

Dans ce projet, j’ai conçu un site web complet pour un parc de loisirs fictif nommé DinoTerra, un lieu immersif autour de l’univers des dinosaures. Ce travail avait pour but de développer un site responsive, accessible et avec une identité visuelle forte. J'ai créé plusieurs pages HTML (accueil, activités, contact), chacune avec son contenu spécifique, en veillant à maintenir une cohérence graphique sur l’ensemble du site.

L’interface intègre une barre de navigation fonctionnelle, un formulaire de contact, une carte interactive, des blocs d’activités présentés de manière homogène, ainsi qu’un pied de page informatif. Le site répond aux normes W3C et utilise des requêtes média pour l’adaptation à toutes les tailles d’écran.

Compétences développées :

  • Conception de maquettes et réflexion sur l’identité visuelle d’un site

  • Utilisation des technologies HTML5 et CSS3 (Flexbox, Grid, Media Queries)

  • Création de sites responsive et conformes aux standards W3C

  • Intégration d’éléments multimédias (vidéo, carte interactive, formulaire)

Langage et environnement utilisés :

  • HTML / CSS

  • Outils : Validateur W3C, éditeur de code (VS Code), flaticon, dafont

Création d’un jeu vidéo en 24h lors de la Code Game Jam 2025

Résumé :

Dans le cadre de la Code Game Jam 2025, j’ai participé à un défi de création de jeu vidéo en 24 heures en équipe. L’objectif était de concevoir un jeu complet, de l’idée initiale au produit jouable, dans un temps très restreint. Ce projet a nécessité une grande réactivité, de la créativité et une coordination constante avec les autres membres de l’équipe. Nous avons défini une mécanique de jeu originale et développé un prototype jouable dans Unity avec des graphismes et des mécaniques simples mais efficaces.

Le projet a été géré à l’aide de Git pour assurer un versionnage propre et une collaboration fluide. Nous avons également pris soin d’équilibrer gameplay et esthétique dans un laps de temps court, ce qui a renforcé notre capacité à faire des choix techniques et créatifs rapidement.

Compétences développées :

  • Développement rapide de jeux vidéo avec Unity

  • Programmation en C# appliquée au gameplay

  • Utilisation de Git pour le versionnage et la collaboration en équipe

  • Travail en équipe sous pression avec gestion du temps serrée

  • Création d’une mécanique de jeu originale et équilibrée

Langage et environnement utilisés :

  • Unity

  • C#

  • Git

  • Environnement de développement : Visual Studio / Unity Editor
